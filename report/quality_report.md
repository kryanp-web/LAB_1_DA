# Отчет о качестве данных

## 1. Введение

Данные представляют собой информацию о клиентах банка, связанную с их социально-демографическими характеристиками (возраст, работа, семейное положение, образование) и финансовым положением (баланс, наличие кредитов), а также с результатами маркетинговой кампании (была ли оформлена срочная депозитная ставка) в 2021 году.

Заказчиком анализа выступает маркетинговый отдел банка. Их цель — повысить эффективность маркетинговых кампаний, сократив затраты на привлечение клиентов и увеличив их конверсию.

С помощью этого датасета можно решить такие задачи интеллектуального анализа данных, как:
- Классификация (предсказание, оформит ли клиент депозит);
- Кластеризация / сегментация клиентов;
- Описательная аналитика (выявление ключевых факторов успеха кампании).

## 2. Паспорт датасета

В датасете 4521 строк, 16 столбцов.

**Признаки:**
| Название | Тип | Описание |
|----------|-----|----------|
| age | int64 | Возраст клиента (в годах) |
| job | object | Тип работы (есть значения с пробелами) |
| marital | object | Семейное положение (divorced, married, single) |
| education | object | Уровень образования (primary, secondary, tertiary, unknown) |
| default | object | Есть ли дефолт по кредиту? (yes/no) |
| balance | int64 | Среднегодовой баланс на счету (в евро) |
| housing | object | Есть ли ипотечный кредит? (yes/no) |
| loan | object | Есть ли потребительский кредит? (yes/no) |
| contact | object | Тип контакта с клиентом (cellular, telephone, unknown) |
| day | int64 | День месяца последнего контакта |
| month | object | Месяц последнего контакта (jan, feb, mar, ...) |
| duration | int64 | Длительность последнего контакта (в секундах) |
| campaign | int64 | Количество контактов в рамках этой кампании |
| pdays | int64 | Дней после предыдущего контакта (-1 = не контактировали ранее) |
| previous | int64 | Количество контактов до этой кампании |
| poutcome | object | Результат предыдущей кампании (failure, other, success, unknown) |

## 3. Основные проблемы качества данных

| № | Тип проблемы | Описание | Потенциальный риск для анализа | Предлагаемое решение/ стратегия |
|---|--------------|----------|--------------------------------|---------------------------------|
| 1 | Пропущенные значения | В столбце `poutcome` 1215 пропусков (26.87%). В столбце `contact` 33 пропуска (0.73%). | Высокий риск для анализа эффективности кампаний. Потеря значительной части данных о результатах предыдущих кампаний. | Для `poutcome` (27%) — рассмотреть возможность удаления признака. Для `contact` (0.7%) — удалить строки с пропусками. |
| 2 | Специальные коды в числовых признаках | В столбце `pdays` значение -1 используется для обозначения случаев, когда с клиентом ранее не контактировали. | Значение -1 можно интерпретировать как обычное число, что приведет к некорректным выводам о временных промежутках. | Заменить -1 на пропущенное значение и создать отдельный бинарный признак `was_contacted_before` (True/False). |
| 3 | Аномальные значения в `duration` | В столбце `duration` обнаружены отрицательные значения (22 записи), что физически невозможно. Также есть очень короткие звонки (0 секунд). | Некорректные данные искажают статистику и могут привести к ошибкам в анализе эффективности звонков. | Удалить строки с отрицательной длительностью звонка. Проанализировать звонки нулевой длительности. |
| 4 | Пробелы в категориальных данных | В столбце `job` обнаружено 16 значений с пробелами перед словом (например, " management", " blue-collar"). | Пробелы приводят к созданию нескольких категорий для одной профессии, что искажает анализ распределения профессий. | Очистить текст, удалить пробелы в начале строк. |
| 5 | Аномальные значения в `balance` и `duration` | В `balance` обнаружены отрицательные значения (366 записей) и экстремально высокие значения. В `duration` — аномально высокие значения (до 3025 сек) и отрицательные. | Выбросы могут сильно повлиять на средние значения. | Проанализировать природу выбросов. Отрицательный баланс оставить, так как это бизнес-особенность. |
| 6 | Несогласованность данных | В столбце `duration` есть отрицательные значения (-1721, -297 и др.), что является явной ошибкой ввода данных. | Некорректные данные делают анализ длительности звонков бессмысленным. | Удалить строки с отрицательной длительностью звонка. |

## 4. План улучшения качества данных

**Высокий приоритет:**

- Удаление строк с отрицательной длительностью звонка (`duration < 0`).
- Очистка пробелов в категориальных признаках (`job`).
- Замена специального значения -1 в `pdays` .

**Средний приоритет:**

- Обработка пропусков в `poutcome` (27%): рассмотреть возможность удаления признака.
- Удаление строк с пропусками в `contact` (0.7%).
- Обработка выбросов в `balance` и `duration`.
- Создание даты из `day` и `month` для анализа сезонности.

**Низкий приоритет:**

- Создание производных признаков:
  - Из даты выделить месяц, день недели, квартал для анализа сезонности.
  - Создать признак `balance_category` (отрицательный, низкий, средний, высокий).

## 5. Краткое резюме

В процессе работы научился описывать датасет, выполнять первичный обзор данных, проводить аудит качества данных, строить базовые графики для визуализации, выявлять ошибки и аномалии в данных, а также предлагать пути их решения. Освоил работу с библиотеками pandas, matplotlib и seaborn в среде Jupyter Notebook.

Особого внимания при дальнейшем интеллектуальном анализе данных требуют:
- Высокий процент пропусков (27%) в признаке `poutcome`, что может потребовать его исключения.
- Специальное значение -1 в признаке `pdays`, которое необходимо корректно обработать перед анализом.
- Отрицательные значения в `duration`, которые являются явной ошибкой и должны быть удалены.
- Пробелы в значениях профессий (`job`), которые создают дублирующиеся категории.
